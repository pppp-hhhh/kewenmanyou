# 课文漫游 - 开发分工与接口大纲

## 一、团队分工

| 成员 | 职责 | 核心模块 |
|------|------|----------|
| **lzl** | 后端开发 | AI分析（DeepSeek）、图像生成（ComfyUI）、数据库设计、任务队列、API接口 |
| **zgl** | 前端开发 | 展示广场（作品浏览、观看页）、用户个人中心（Phase 2） |
| **ph** | 前端开发 | 工作台（课文输入、AI分析、场景编辑、漫画生成与预览） |

---

## 二、协作模式

- **后端（lzl）** 提供 RESTful API，部署在 `http://localhost:8000`（开发环境）。
- **前端（zgl + ph）** 基于 Nuxt 4 开发，`app/` 目录下各自开发独立页面，通过 API 与后端交互。
- **前端项目**：`app/` 结构（`app/pages/`, `app/components/`, `app/stores/`, `app/composables/`）
- **后端项目**：`server/` 目录（独立 FastAPI）
- **接口文档**：Swagger（FastAPI 自动生成，`http://localhost:8000/docs`）+ 共享 Markdown。
- **版本控制**：Git 单仓库，分支策略：
  - `main`：稳定分支
  - `dev-backend`：后端开发分支
  - `dev-frontend-ph`：工作台开发分支
  - `dev-frontend-zgl`：展示广场开发分支
- **联调**：每周两次同步会议，根据 API 进度调整前端调用。

## 三、后端（lzl）详细任务

### 3.1 数据库设计（SQLite → PostgreSQL 可选）

**表结构**（MVP 先实现 `texts` 和 `user_works`，用户表后续可加）

#### `texts`（内置课文库）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| title | VARCHAR(200) | 标题 |
| author | VARCHAR(100) | 作者 |
| content | TEXT | 全文 |
| grade | VARCHAR(20) | 年级 |
| category | VARCHAR(20) | 古诗/文言文/现代文 |
| created_at | DATETIME | |

#### `user_works`（用户作品）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| user_id | INTEGER | 暂时可空（未登录用户保存时填0） |
| text_id | INTEGER | 关联 texts.id，可为空 |
| custom_title | VARCHAR(200) | |
| custom_content | TEXT | 用户输入的课文原文 |
| scenes | JSON | 场景列表 [{description_cn, prompt_en}] |
| images | JSON | 图片 URL 数组（相对路径） |
| style | VARCHAR(50) | 画风 |
| thumbnail | VARCHAR(255) | 缩略图路径 |
| is_public | BOOLEAN | 默认0 |
| view_count | INTEGER | 默认0 |
| created_at | DATETIME | |

**索引**：
- `idx_user_works_user_id`
- `idx_user_works_is_public`
- `idx_user_works_created_at`

### 3.2 API 接口清单

#### 3.2.1 课文相关
| 方法 | 路径 | 功能 | 备注 |
|------|------|------|------|
| GET | `/api/texts` | 获取内置课文列表（分页、按年级/分类筛选） | 供工作台选择课文 |
| GET | `/api/texts/{id}` | 获取课文详情 | |

#### 3.2.2 AI 分析与生成
| 方法 | 路径 | 功能 | 备注 |
|------|------|------|------|
| POST | `/api/analyze` | 调用 DeepSeek 分析课文，返回场景列表 | 请求：`{text, style}`；响应：`{scenes: [{description_cn, prompt_en}]}` |
| POST | `/api/generate` | 提交批量生成任务 | 请求：`{prompts: string[], style}`；响应：`{task_id}` |
| GET | `/api/task/{task_id}` | 查询任务状态（轮询） | 响应：`{status, total, completed, images: [{index, url}], error}` |
| DELETE | `/api/task/{task_id}` | 取消任务（可选） | |

#### 3.2.3 作品保存与展示
| 方法 | 路径 | 功能 | 备注 |
|------|------|------|------|
| POST | `/api/works` | 保存作品（用户保存当前生成的漫画） | 请求体含 scenes, images, style 等；响应 `{work_id}` |
| GET | `/api/works/public` | 获取公开作品列表（分页、筛选） | 供展示广场使用 |
| GET | `/api/works/{work_id}` | 获取作品详情（含所有场景和图片） | 供观看页使用 |
| PUT | `/api/works/{work_id}` | 更新作品（公开/私密、删除等） | 后续 |
| GET | `/api/works/download/{work_id}` | 下载作品 ZIP 包 | 返回 ZIP 文件 |

#### 3.2.4 用户相关（Phase 2）
- 注册/登录（JWT）
- 我的作品列表
- 收藏夹

### 3.3 后端技术实现要点

- **框架**：Python 3.10, FastAPI + Uvicorn
- **任务管理**：内存字典 `tasks` + `BackgroundTasks`（不使用 Celery，减少依赖）
- **并发控制**：`asyncio.Semaphore` 限制同时生成的图片数量（默认 2）
- **图像生成**：封装 ComfyUI 的 API 调用，支持 LCM LoRA 加速（目标 8-10 秒/张）
- **分辨率**：512×512 或 512×384
- **图片存储**：
  - 临时目录：`./static/temp/`（任务完成后保留 1 小时，定时清理）
  - 作品目录：`./static/works/{work_id}/`（保存后移入）
- **缩略图**：保存作品时自动生成第一张图片的 200x150 缩略图
- **DeepSeek 调用**：使用 `openai` 库（base_url 指向 DeepSeek），添加超时和重试
- **缓存**：对于相同课文内容（哈希）的分析结果，可缓存 1 天（避免重复计费）

### 3.4 环境变量（.env）
```
DEEPSEEK_API_KEY=your_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
COMFYUI_API_URL=http://localhost:8188/api/generate
MAX_CONCURRENT_GENERATIONS=2
TEMP_IMAGE_DIR=./static/temp
WORKS_IMAGE_DIR=./static/works
DATABASE_URL=sqlite:///./app.db
```

### 3.5 启动命令
```bash
cd server
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

## 四、前端工作台（ph）详细任务

### 4.1 技术栈
- **框架**：Nuxt 4 + Vue 3
- **状态管理**：Pinia（`stores/workspace.ts`）
- **样式**：Tailwind CSS + Nuxt UI
- **图标**：nuxt-icon
- **渲染模式**：CSR（客户端渲染）

### 4.2 页面路由
- `/workspace` —— 工作台主页面

### 4.3 页面布局（左右分栏）
- **左侧设置区**：
  - 课文来源（下拉选择内置课文 / 自定义文本框）
  - 画风选择（写实古风、水墨风格、彩色插画）
  - 按钮「AI 分析课文」
  - 场景编辑列表（每个场景显示中文描述，可编辑、增删、上移/下移）
  - 按钮「开始生成」
  - 实时提示区（分析进度、生成进度、剩余时间）
- **右侧预览区**：
  - CSS Grid 布局（每行 3~4 张，响应式）
  - 生成过程中每张图片由灰色占位 → 显示实际图片，带加载动画
  - 鼠标悬浮显示中文描述
  - 点击图片放大查看
  - 全部生成完成后出现「保存作品」按钮

### 4.4 交互流程

1. 用户选择课文或粘贴自定义课文。
2. 点击「AI 分析课文」→ 调用 `POST /api/analyze` → 返回场景列表 → 渲染到编辑区。
3. 用户可编辑场景中文描述（修改后自动同步生成英文 prompt，或暂时不支持自动同步，提示用户重新分析）。
4. 点击「开始生成」→ 调用 `POST /api/generate`（提交 prompts 数组和 style）→ 获得 `task_id`。
5. 启动轮询 `GET /api/task/{task_id}`（间隔 1 秒）：
   - 每次收到响应，如果 `images` 数组比之前长，则将新增的图片渲染到右侧网格中的对应位置（根据 `index`）。
   - 更新进度提示（`completed/total`），计算剩余时间（假设每张 10 秒）。
6. 所有图片完成后，显示「保存作品」按钮。
7. 点击保存 → 调用 `POST /api/works`（传递当前作品数据）→ 成功后可跳转到个人中心或展示页。

### 4.5 状态管理（Pinia stores/workspace.ts）
- `selectedTextId`（课文内容）
- `customText`（自定义文本）
- `selectedStyle`（画风）
- `scenes`（场景列表）
- `taskId`, `taskStatus`, `generatedImages`
- `isGenerating`, `progressMsg`

### 4.6 Composables
- `useApiFetch` — API 请求封装
- `useAnalyze` — 课文分析逻辑
- `useGenerate` — 任务提交与轮询
- `useWorks` — 作品保存

### 4.7 关键技术点
- **轮询取消**：组件卸载或新任务开始时清除之前的 `setInterval`。
- **图片懒加载**：使用 `<img loading="lazy">`。
- **下载 ZIP**：调用 `GET /api/works/download/{work_id}` 触发浏览器下载。
- **移动端适配**：工作台在手机屏幕可能需折叠左侧区域（使用 `drawer`），但 MVP 可先保证 PC。

## 五、前端展示广场（zgl）详细任务

### 5.1 技术栈
- **框架**：Nuxt 4 + Vue 3
- **状态管理**：Pinia（`stores/gallery.ts`）
- **样式**：Tailwind CSS + Nuxt UI
- **图标**：nuxt-icon
- **渲染模式**：SSR/SSG（服务端渲染/静态生成）

### 5.2 页面路由
- `/gallery` —— 展示广场（作品列表）
- `/watch/{id}` —— 观看页（漫画阅读器）

### 5.3 展示广场（作品列表）

**布局**：
- 顶部筛选栏：按画风下拉、按年级下拉、搜索框（课文标题）。
- 作品网格：每行 4~5 个卡片，每个卡片包含：
  - 缩略图（若未生成缩略图则显示占位图）
  - 标题（用户自定义标题或课文名）
  - 作者（或"用户"）
  - 画风标签
  - 浏览量
- 底部分页组件（上一页/下一页，跳转）。

**交互**：
- 页面加载时调用 `GET /api/works/public?page=1&page_size=12`。
- 筛选条件变化时重新请求。
- 点击卡片跳转到 `/watch/{id}`。

### 5.4 观看页（漫画阅读器）

**布局**：
- 顶部：作品标题、作者、画风、浏览量、返回按钮。
- 中间：漫画网格展示（每行 4 张，可切换为单页滑动模式，优先做网格）。
- 每个图片下方显示场景序号，鼠标悬浮显示中文描述。
- 点击任意图片弹窗放大（简单模态框）。
- 可选功能：播放全部 TTS（调用浏览器 `speechSynthesis` 逐句朗读中文描述），提供播放/暂停按钮。

**交互**：
- 进入页面调用 `GET /api/works/{work_id}` 获取作品详情（包含 scenes 和 images）。
- 渲染所有图片和描述。
- 如果作品未公开且未登录，提示无权访问（但 Phase 1 可先不做权限控制）。

### 5.5 个人中心（Phase 2）
- 我的作品列表（管理：编辑、删除、公开/私密切换）
- 收藏夹

### 5.6 技术要点
- 图片网格使用 CSS Grid，响应式。
- TTS 使用 `window.speechSynthesis`，注意移动端需要用户交互才能播放。
- 作品详情可缓存到本地（localStorage）减少重复请求。

## 六、前后端接口详细定义（供联调）

### 6.1 分析课文 `POST /api/analyze`
**请求**
```json
{
  "text": "从百草园到三味书屋...",
  "style": "写实古风"
}
```
**响应**（成功）
```json
{
  "scenes": [
    {
      "description_cn": "碧绿的菜畦，光滑的石井栏，高大的皂荚树...",
      "prompt_en": "lush green vegetable garden, smooth ancient stone well... realistic ancient Chinese style"
    }
  ]
}
```
**错误**：`{"detail": "分析失败，请重试"}`

### 6.2 提交生成 `POST /api/generate`
**请求**
```json
{
  "prompts": ["prompt1", "prompt2"],
  "style": "写实古风"
}
```
**响应**
```json
{
  "task_id": "abc123"
}
```

### 6.3 查询任务 `GET /api/task/{task_id}`
**响应**（处理中）
```json
{
  "status": "processing",
  "total": 6,
  "completed": 2,
  "images": [
    {"index": 0, "url": "/static/temp/abc123_0.png"},
    {"index": 1, "url": "/static/temp/abc123_1.png"}
  ],
  "error": null
}
```
**响应**（完成）
```json
{
  "status": "completed",
  "total": 6,
  "completed": 6,
  "images": [...],
  "error": null
}
```

### 6.4 保存作品 `POST /api/works`
**请求**
```json
{
  "text_id": 1,                     // 可选
  "custom_title": "我的范进中举",
  "custom_content": "课文原文...",  // 可选，如无 text_id 则必填
  "scenes": [...],                 // 完整的场景列表
  "images": ["url1", "url2"],      // 图片相对路径（从 task 中获取）
  "style": "写实古风",
  "is_public": true
}
```
**响应**
```json
{
  "work_id": 123,
  "message": "保存成功"
}
```

### 6.5 公开作品列表 `GET /api/works/public`
**参数**：`page`（默认1）、`page_size`（默认12）、`style`（可选）、`keyword`（可选，标题搜索）
**响应**
```json
{
  "total": 100,
  "page": 1,
  "page_size": 12,
  "items": [
    {
      "work_id": 123,
      "title": "《范进中举》动漫版",
      "thumbnail": "/static/works/123/thumb.jpg",
      "style": "写实古风",
      "view_count": 45,
      "created_at": "2025-03-20T10:00:00Z"
    }
  ]
}
```

### 6.6 作品详情 `GET /api/works/{work_id}`
**响应**
```json
{
  "work_id": 123,
  "title": "《范进中举》动漫版",
```

## 九、验收与交付

| 编号 | 验收项 | 标准 |
|------|--------|------|
| 1 | 课文分析 | ≤3000字课文，10秒内返回动态场景列表（6~30个），关键细节齐全 |
| 2 | 场景编辑 | 支持增删、移动、修改中文描述 |
| 3 | 生成速度 | 单张图片 ≤12秒（含网络和推理），并发2张时吞吐量稳定 |
| 4 | 边生成边看 | 生成过程中右侧网格增量显示图片，无需等待全部完成 |
| 5 | 图片还原度 | 人物外貌、服装、场景、情节符合课文，至少3位语文老师/学生评估平均分≥4（5分制） |
| 6 | 保存下载 | 可保存作品到云端（或下载ZIP），ZIP含所有图片和JSON元数据 |
| 7 | 展示广场 | 能浏览公开作品列表，能进入观看页看图读文字 |
| 8 | 稳定性 | 连续生成3次（每次10格以上）无报错，无显存溢出 |

## 十、风险与应对

| 风险 | 应对措施 |
|------|----------|
| GPU显存不足导致崩溃 | 限制并发数为1，降低分辨率到448×448，换用更轻量模型 |
| DeepSeek API超时或费用超支 | 缓存相同课文分析结果，限制单用户每日调用次数 |
| 图片还原度不达预期 | 迭代DeepSeek提示词工程，增加"不准确"用户反馈收集数据 |
| 用户等待时间仍感觉长 | 增加剩余时间显示；提供"优先生成前6格"快速预览 |
| Nuxt 4 生态尚新，部分模块兼容性问题 | 锁定模块版本，参考Nuxt 4迁移指南，保留降级方案 |
